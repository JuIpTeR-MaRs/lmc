# 🗳️ 多阶段去中心化投票系统 (Voting DApp)

基于以太坊区块链的公平、公正、透明的多阶段投票系统。项目前端采用 Vanilla HTML/JS 与 ethers.js 交互，智能合约采用 Solidity 开发，使用 Hardhat 框架编译、部署和进行本地链上模拟。

---

## 🛠️ 前提条件

运行和测试本项目，你需要准备：
1. **Node.js** (推荐 v18 或更高版本)
2. **MetaMask 钱包插件** (已安装在你的 Chrome / Edge 等浏览器上)

---

## 🚀 部署与运行步骤

项目目录下提供了两个一键运行的批处理脚本，双击即可完成部署和启动：

### 步骤 1：启动本地节点并部署合约
1. 双击运行项目目录下的 **`run_deploy.bat`**。
2. 命令行窗口弹出后，输入数字 **`1`**（即选择 `Auto-Start Node and Deploy to Localhost`）并回车。
3. 脚本会自动：
   * 在新窗口中启动 Hardhat 本地开发节点（监听 `http://127.0.0.1:8545`，**请保持该新窗口开启，不要关闭**）。
   * 自动编译 Solidity 智能合约。
   * 将合约部署到本地开发链，并**自动把最新部署的合约地址写入到 `index.html` 中**。

### 步骤 2：启动网页服务器
1. 双击运行项目目录下的 **`run_frontend.bat`**。
2. 脚本会在本地 `8080` 端口启动 Web 服务器，并**自动用你的默认浏览器打开页面**：
   * 🔗 [http://localhost:8080](http://localhost:8080)
3. ⚠️ **注意**：**请不要**在文件管理器中直接双击 `index.html` 运行！因为浏览器对本地文件协议（`file://`）有安全沙箱限制，会导致 MetaMask 钱包无法正常连接和注入。必须通过上述的 `localhost:8080` 网址访问。

## 📝 完整使用教程

本系统包含多个角色（拥有者/管理员/审核员/普通选民）和多个投票阶段：

### 1. 角色与权限管理 (需要连接 Owner 账户)
* **添加管理员/审核员**：切换到 Owner 账号，进入“权限管理”选项卡。输入你想授权的钱包地址，点击“添加管理员”或“添加审核员”。
* **自动按钮保护**：只有当前连接的账号具有对应权限时，操作按钮才会解锁点亮，非授权账号按钮将被自动灰显禁用。

### 2. 创建投票活动 (需要管理员权限)
1. 切换到管理员账户，进入“创建投票活动”选项卡。
2. 填写活动标题、描述、投票开始与结束时间、候选人上限。
3. 选择 **投票模式**：
   * **1人1票**：每个账户只有 1 票权重。
   * **按代币权重投票**：根据账户里持有的 VGT 投票代币数量决定投票权重。
4. 点击“创建活动”并发送链上交易。

### 3. 申请成为候选人
1. 点击列表中的活动卡片查看详情。
2. 在“申请成为候选人”板块中，输入候选人的“姓名”和“简介”，点击提交并发送交易。
3. 申请提交后，该候选人会处于 **`待审核`** 状态，普通选民还无法对其进行投票。

### 4. 审核候选人 (需要审核员/管理员权限)
1. 使用具有审核员/管理员权限的账户打开活动详情。
2. 待审核的候选人右下方会显示绿色的 **“通过”** 和红色的 **“驳回”** 按钮。
3. 点击“通过”并发送交易，候选人状态将更新为 **`已通过`**，可以开始接受投票。

### 5. 进行投票 (普通选民)
1. 切换为任意普通的测试账户。
2. 如果是**“代币权重投票”**：点击页面顶部的 **“领取代币 (Faucet)”** 按钮免费领取 100 VGT 投票代币。
3. 在活动详情中，找到“已通过”状态的候选人，点击 **“投票”**，并在 MetaMask 中确认即可。
4. 页面底部的 **“投票溯源 / 链上记录”** 栏目会自动刷新，实时渲染出包含投票人、投给对象、所占权重及具体时间的链上区块记录，做到数据公开透明不可篡改。

## ❓ 常见问题排查 (FAQ)

### Q: 页面提示“错误：该地址上没有部署合约！”？
1. 请检查 MetaMask 顶部当前选择的网络是否是 `Localhost 8545`。如果是 Sepolia 或以太坊主网，由于读不到本地合约，会报错。
2. 检查 Hardhat 本地节点命令行窗口是否被关闭了，若关闭了请重新运行 `run_deploy.bat` 部署合约，并按 `Ctrl + F5` 强制刷新网页清除浏览器缓存。

### Q: MetaMask 提示“查看提醒”，并且网络费处有警告标志，无法直接确认？
1. 这是由于你使用的是未加密的 `http` 协议局域网 IP（例如 `198.18.0.x`）访问网页，MetaMask 会进行安全阻断保护。
2. **解决方法**：点击“查看提醒”滚动到最下方确认以跳过风险警示；或者在浏览器地址栏将网址改为 `http://localhost:8080` 访问，本地回环地址不会触发该警告。

### Q: MetaMask 提示“交易可能失败”或“无法估算 Gas 费”？
1. **本地时间差问题**：本地链的时间只有在产生交易时才会推进。如果设置的投票开始时间还未到达（比如由于本地节点闲置导致链上时间比实际时间滞后），交易就会失败。可以先点一次顶部的“领取代币 (Faucet)”发送一笔交易来刷新推进链上时间。
2. **Nonce 冲突问题**：如果你重启了 Hardhat 本地链，链的交易笔数清零了，但 MetaMask 还记录着旧的交易数。
   * **解决方法**：打开 MetaMask -> 点击“设置” -> “高级” -> 点击 **“清除活动和随机数历史记录”**（或“重置账户”）即可解决。
